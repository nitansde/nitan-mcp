Nitan MCP 项目主页

连接论坛能力与智能体工作流

Nitan 是基于 Discourse MCP 深度改造的项目，面向 uscardforum 使用场景，提供稳定的 MCP 工具能力，并同时发布可复用的 nitan Skill。

GitHub 链接（占位） NPM 包页面 uscardforum 讨论贴

项目简介

Nitan 项目用于把 Discourse 论坛能力暴露为 MCP 工具，便于 Claude Desktop 等客户端按统一接口完成搜索、阅读与自动化流程。项目默认以只读模式安全运行，也支持在满足权限条件下开启写入能力。

该仓库同时提供 Skill（名称为 nitan）用于增强智能体执行体验，Skill 为附加能力，不改变 MCP 运行时行为。

核心能力

论坛数据工具：搜索、读取话题、读取帖子、分类与标签浏览。
站点模式：支持固定站点模式或动态选择站点模式。
权限控制：默认只读，写入能力由参数和鉴权共同决定。
抗防护能力：Cloudflare 双策略自动回退，提升请求稳定性。
远程工具扩展：可按站点发现并注册远程工具。
Skill 支持：可打包并分发 nitan Skill，支持本地或文件导入。

快速开始

  1. 准备环境：Node.js 18+，Python 3.7+，以及 pip3。
  2. 推荐直接运行：npx -y @nitansde/mcp@latest。
  3. 如需本地开发，执行以下命令：
pnpm install
pnpm build
node dist/index.js

如 Python 依赖未自动安装，可执行 pip3 install -r requirements.txt。

Skill 安装

Skill 名称为 nitan，可通过打包产物导入。当前准确产物路径为 dist-skill/nitan.skill。 

pnpm skill:pack
# 输出：dist-skill/nitan.skill
  • 方式一（推荐）：导入 dist-skill/nitan.skill 到 Skill 管理器。
  • 方式二：复制 skills/nitan/ 到本地 Skill 目录并重载。

MCP 客户端配置示例

以下示例展示名称为 nitan 的 MCP 服务器配置。

{
  "mcpServers": {
    "nitan": {
      "command": "npx",
      "args": ["-y", "@nitansde/mcp@latest"],
      "env": {
        "NITAN_USERNAME": "你的用户名（可选）",
        "NITAN_PASSWORD": "你的密码（可选）",
        "TIMEZONE": "America/New_York"
      }
    }
  }
}

说明：NITAN_USERNAME 与 NITAN_PASSWORD 为可选变量；不填写时可使用公开只读能力。TIMEZONE 用于指定时区，未设置时默认使用本地时钟。

常见问题

问：不配置账号密码可以用吗？

答：可以，公开内容的只读访问可直接使用；需要受限能力时再配置账号密码。

问：Cloudflare 相关功能失效怎么办？

答：先确认 Python 依赖已安装，再检查运行环境中的 Python 路径是否可用。

问：Skill 会改变 MCP 服务行为吗？

答：不会，Skill 为附加能力，MCP 运行逻辑保持不变。
